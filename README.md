## Setup

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Place model artefacts
Ensure the `model_artefacts_v2/` folder (generated by the notebook's Phase 7) is in the same directory as `app.py`:

```
app.py
requirements.txt
model_artefacts_v2/
  ├── xgb_model.pkl
  ├── scaler.pkl
  ├── label_encoder.pkl
  ├── app_tier_map.pkl
  ├── keywords.pkl
  ├── feature_columns.pkl
  └── class_indices.pkl
```

### 3. Run the app
```bash
streamlit run app.py
```

## Features

| Feature | Description |
|---|---|
| **LLM semantic extraction** | Calls Claude Haiku to extract 10 semantic signals from the observation and risk text |
| **XGBoost scoring** | Loads the trained XGBoost model to produce a 0–100 continuous risk score |
| **Risk band classification** | Critical (≥75), High (≥55), Medium (≥35), Low (<35) |
| **Class probabilities** | P(High), P(Medium), P(Low) displayed as visual bars |
| **SHAP explainability** | Top-10 feature attributions rendered as a horizontal bar chart |
| **Cyber risk flags** | 6 structured keyword-based flags from Phase 2a |
| **LLM signal grid** | All 10 Claude Haiku semantic features shown with active/inactive indicators |
| **Session history** | Last 5 scored findings shown in the sidebar |
| **JSON export** | Download full scoring result as JSON (Aura-compatible) |
| **Quick examples** | Pre-loaded High / Medium / Low severity examples for testing |

## API Key

Enter your Anthropic API key in the sidebar. The app uses `claude-haiku-4-5-20251001` for LLM feature extraction (~$0.001 per finding). If no key is provided, the app will use neutral fallback defaults for the LLM features.
